JPublish StringTemplate component DEMO page

JPublish application demonstrating a Component level integration with the StringTemplate (ST) engine.

This page path: $page.Path, is mostly driven by the current JPublish View Renderer: $site.ViewRenderer, but with the ability to use ST template files.

For example, using Velocity syntax you can define the following Velocity variables:
\#set( $column = "USER_NAME")
\#set( $table = "USER_PROFILES")
#set( $column = "USER_NAME") #set( $table = "USER_PROFILES")
And then use them in a ST template rendered by the JPublish ST component with a simple command like this:
\$components.JPST.renderPath("index"), or: \$components.JPST.renderPath("default:index")

The original ST file looks like this: 
-- a basic query statement
select \$column$ from \$table$;
GO
and the rendered result looks like this: 
$components.JPST.renderPath("index")
Including a template from a different ST group is as easy as specifying the group name:
\$components.JPST.renderPath("menus:main_menu"), where "menus" represents the ST group defined in your jpublish.xml file. 
$components.JPST.renderPath("menus:main_menu")
